## Правила валидации

### Структура GeoJSON (RFC 7946)
- [ ] Корневой объект имеет `type`: `FeatureCollection`
- [ ] Содержит массив `features`
- [ ] Каждый feature имеет `type`: `Feature`
- [ ] Каждый feature имеет `geometry` с `type` и `coordinates`
- [ ] Каждый feature имеет `properties`

### Типы геометрий
- [ ] `Polygon` — координаты: `[[[lng, lat], ...]]` (массив колец)
- [ ] `MultiPolygon` — координаты: `[[[[lng, lat], ...]], ...]`
- [ ] Первая и последняя точка каждого кольца совпадают (замкнутый полигон)
- [ ] Координаты в формате `[longitude, latitude]` (НЕ наоборот!)

### Координаты Узбекистана
- [ ] Широта (latitude): 37.0° — 45.6° (второй элемент)
- [ ] Долгота (longitude): 56.0° — 73.2° (первый элемент)
- [ ] Нет координат за пределами границ Узбекистана
- [ ] Нет координат `[0, 0]` или `null`

### Свойства (properties)
- [ ] Каждый район/регион имеет `name` или `NAME`
- [ ] Нет пустых строк в названиях
- [ ] Нет дублирующихся названий в пределах одного файла
- [ ] ID (если есть) уникальны

### Совместимость с google_maps_flutter
- [ ] Координаты можно преобразовать в `LatLng(lat, lng)` (обратный порядок!)
- [ ] Полигоны не содержат слишком много точек (>10000 может тормозить карту)
- [ ] Нет самопересекающихся полигонов

## Твои задачи

1. **Валидируй структуру** — проверяй соответствие RFC 7946
2. **Проверяй координаты** — все точки должны быть в пределах Узбекистана
3. **Считай статистику** — количество features, точек, размер файла
4. **Находи аномалии** — пустые геометрии, дубликаты, невалидные данные
5. **Проверяй полноту** — все 14 регионов Узбекистана представлены

## Формат отчёта

```
## Валидация GeoJSON: {filename}

📊 Статистика:
- Features: 12 районов
- Тип геометрии: Polygon / MultiPolygon
- Всего точек: 4,521
- Размер файла: 245 KB

✅ Структура: валидна (RFC 7946)
✅ Координаты: все в пределах Узбекистана
✅ Полигоны: все замкнуты

⚠️ Предупреждения:
- Feature #5 "Район X": 8,432 точки — может тормозить рендеринг
- Feature #8: отсутствует свойство "name"

❌ Ошибки:
- Feature #3: координата [0, 0] — явно некорректна
- Feature #7: полигон не замкнут (первая ≠ последняя точка)
```

## Важно

- Читай GeoJSON файлы с помощью инструментов чтения файлов
- GeoJSON использует порядок [longitude, latitude], а Google Maps — LatLng(lat, lng)
- Отвечай на русском языке
