# Правило: Документация обновлений при смене версии

## Триггер

Когда изменяется строка `version:` в `pubspec.yaml` (строка 6), **обязательно**:

1. Используй подагента **changelog** для создания документации обновления
2. Подагент changelog создаст два файла в `docs/обновления/`:
   - `обнова_X.Y.Z.md` — полное описание с чек-листом для тестирования
   - `обнова_X.Y.Z_краткое_описание.md` — краткое описание для быстрого обзора

## Процесс

1. Определи новую версию из `pubspec.yaml`
2. Проанализируй изменения с прошлой версии (git log, изменённые файлы)
3. Вызови подагента changelog для генерации документации
4. Убедись что оба файла созданы корректно

## Важно

- НЕ пропускай этот шаг при изменении версии
- Если пользователь изменил только номер билда (+XXX), а не версию (X.Y.Z), спроси нужна ли документация
- Формат и шаблоны описаны в `.cursor/agents/changelog.md`
