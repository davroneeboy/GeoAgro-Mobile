## 1. CeyTrack — Обратная геокодировка + GPS-трекинг + Карта земель

**Репозиторий:** https://github.com/malitha-tishamal/CeyLogix-Field-Tracking-App
**Стек:** Flutter + Firebase + GIS + Google Maps
**Лицензия:** Open Source

### Что внедрить:

#### a) Обратная геокодировка (адрес по координатам) с 3 уровнями fallback:
1. Пакет `geocoding` (быстрый, локальный)
2. Nominatim API от OpenStreetMap (бесплатный, подробный)
3. Показ координат как fallback
4. Кэширование адресов (без повторных запросов по тем же координатам)

#### b) Real-time GPS-трекинг:
- Обновление местоположения каждые 5 метров
- Подсчёт пройденного расстояния
- История перемещений (до 100 точек)
- Вибрация телефона при обновлении локации
- Переключатель вкл/выкл трекинга

#### c) Карта всех земельных участков:
- Маркеры всех зарегистрированных земель на карте
- Поиск по имени, адресу, культуре, району
- Фильтрация по области/региону
- Просмотр фотографий земельных участков
- Переключение слоёв карты
- Детали земли (культура, размер, владелец, контакт)

#### d) Поиск адреса по тексту:
- Ввод "Ташкент" → карта перемещается туда
- Используется Nominatim API (бесплатный)

### Необходимые пакеты:
```yaml
# Уже есть в проекте:
# geolocator, google_maps_flutter, http, image_picker, url_launcher

# Нужно добавить:
geocoding: ^3.0.0          # Обратная геокодировка
vibration: ^2.0.0          # Вибрация при обновлении GPS (опционально)
```

### Ключевые файлы для изучения:
- `ceytrack/lib/land_owner/land_location.dart` (~2000 строк) — геокодировка, трекинг, поиск
- `ceytrack/lib/factory_owner/lands_map.dart` (~1900 строк) — карта всех земель с фильтрами

### Сложность: Средняя
Код Firebase нужно заменить на наш REST API, но вся логика геокодировки и трекинга готова к копированию.

## 2. Распознавание болезней растений через Gemini AI

**API:** Google Gemini 2.5 Flash-Lite (бесплатный тариф)
**Документация:** https://ai.google.dev/gemini-api/docs/pricing

### Почему Gemini Flash-Lite:
- **БЕСПЛАТНО** — 1 000 запросов в день (хватает на 200 сотрудников × 5 сканов)
- Понимает узбекский и русский язык
- Может давать рекомендации по лечению текстом
- Простая интеграция (один HTTP-запрос с фото)

### Расчёт для 200 сотрудников:

| Сценарий | Запросов/день | Стоимость |
|----------|---------------|-----------|
| 200 чел × 3 скана | 600 | **$0 (бесплатно)** |
| 200 чел × 5 сканов | 1 000 | **$0 (бесплатно, на лимите)** |
| Если превысим лимит | 22 000/мес | ~$5/месяц (платный тариф) |

### Бесплатные лимиты Gemini 2.5 Flash-Lite:
- 15 запросов в минуту (RPM)
- 1 000 запросов в день (RPD)
- 250 000 токенов в минуту (TPM)
- Лимит сбрасывается в полночь по тихоокеанскому времени

### Платный тариф (если понадобится):
- Ввод: $0.10 за 1М токенов
- Вывод: $0.40 за 1М токенов
- ~$0.0002 за один запрос с фото

### Как работает:
```
Сотрудник фоткает растение → Фото отправляется в Gemini API
→ AI анализирует → Возвращает:
  - Название болезни
  - Описание симптомов
  - Рекомендации по лечению
  - Профилактические меры
```

### Необходимые пакеты:
```yaml
google_generative_ai: ^0.4.0   # Gemini AI SDK для Flutter
```

### Примерный запрос к API:
```dart
final model = GenerativeModel(
  model: 'gemini-2.5-flash-lite',
  apiKey: 'YOUR_API_KEY',
);

final prompt = TextPart(
  'Ты агроном-эксперт. Проанализируй фото растения. '
  'Определи болезнь, опиши симптомы и дай рекомендации по лечению. '
  'Ответь на русском языке.'
);

final imagePart = DataPart('image/jpeg', imageBytes);

final response = await model.generateContent([
  Content.multi([prompt, imagePart])
]);
```

### Получение API ключа:
1. Зайти на https://aistudio.google.com/apikey
2. Нажать "Get API key"
3. Создать ключ (бесплатно)

### Важные замечания:
- На бесплатном тарифе Google МОЖЕТ использовать данные для обучения модели
- Если нужна конфиденциальность — платный тариф ($5/мес) решает эту проблему
- Лимит 1000 запросов/день — на ВЕСЬ проект (один API ключ), не на пользователя
- Нужно реализовать очередь запросов, чтобы не превышать 15 RPM

### Сложность: Низкая
Интеграция занимает 1-2 дня. Основная работа — UI экрана камеры и отображение результатов.

## Альтернативы (на будущее):

- **Plant.id API** — специализированный API для растений (~€660/мес для 200 чел) — дорого
- **TFLite на устройстве** — полностью бесплатно и оффлайн, но сложнее в интеграции
- **Модуль погоды** (OpenWeatherMap) — бесплатный тариф, полезно для фермеров
- **Графики/статистика** (fl_chart) — из проекта Agrosmart-flutter
