## 1. CeyTrack — Обратная геокодировка + GPS-трекинг + Карта земель

**Репозиторий:** https://github.com/malitha-tishamal/CeyLogix-Field-Tracking-App
**Стек:** Flutter + Firebase + GIS + Google Maps
**Лицензия:** Open Source

### Что внедрить:

#### a) Обратная геокодировка (адрес по координатам) с 3 уровнями fallback:
1. Пакет `geocoding` (быстрый, локальный)
2. Nominatim API от OpenStreetMap (бесплатный, подробный)
3. Показ координат как fallback
4. Кэширование адресов (без повторных запросов по тем же координатам)

#### b) Real-time GPS-трекинг:
- Обновление местоположения каждые 5 метров
- Подсчёт пройденного расстояния
- История перемещений (до 100 точек)
- Вибрация телефона при обновлении локации
- Переключатель вкл/выкл трекинга

#### c) Карта всех земельных участков:
- Маркеры всех зарегистрированных земель на карте
- Поиск по имени, адресу, культуре, району
- Фильтрация по области/региону
- Просмотр фотографий земельных участков
- Переключение слоёв карты
- Детали земли (культура, размер, владелец, контакт)

#### d) Поиск адреса по тексту:
- Ввод "Ташкент" → карта перемещается туда
- Используется Nominatim API (бесплатный)

### Необходимые пакеты:

## 2. Распознавание болезней растений через Gemini AI

**API:** Google Gemini 2.5 Flash-Lite (бесплатный тариф)
**Документация:** https://ai.google.dev/gemini-api/docs/pricing

### Почему Gemini Flash-Lite:
- **БЕСПЛАТНО** — 1 000 запросов в день (хватает на 200 сотрудников × 5 сканов)
- Понимает узбекский и русский язык
- Может давать рекомендации по лечению текстом
- Простая интеграция (один HTTP-запрос с фото)

### Расчёт для 200 сотрудников:

| Сценарий | Запросов/день | Стоимость |
|----------|---------------|-----------|
| 200 чел × 3 скана | 600 | **$0 (бесплатно)** |
| 200 чел × 5 сканов | 1 000 | **$0 (бесплатно, на лимите)** |
| Если превысим лимит | 22 000/мес | ~$5/месяц (платный тариф) |

### Бесплатные лимиты Gemini 2.5 Flash-Lite:
- 15 запросов в минуту (RPM)
- 1 000 запросов в день (RPD)
- 250 000 токенов в минуту (TPM)
- Лимит сбрасывается в полночь по тихоокеанскому времени

### Платный тариф (если понадобится):
- Ввод: $0.10 за 1М токенов
- Вывод: $0.40 за 1М токенов
- ~$0.0002 за один запрос с фото

### Как работает:
```
Сотрудник фоткает растение → Фото отправляется в Gemini API
→ AI анализирует → Возвращает:
  - Название болезни
  - Описание симптомов
  - Рекомендации по лечению
  - Профилактические меры
```

### Необходимые пакеты:
```yaml
google_generative_ai: ^0.4.0   # Gemini AI SDK для Flutter
```

### Примерный запрос к API:
```dart
final model = GenerativeModel(
  model: 'gemini-2.5-flash-lite',
  apiKey: 'YOUR_API_KEY',
);

final prompt = TextPart(
  'Ты агроном-эксперт. Проанализируй фото растения. '
  'Определи болезнь, опиши симптомы и дай рекомендации по лечению. '
  'Ответь на русском языке.'
);

final imagePart = DataPart('image/jpeg', imageBytes);

final response = await model.generateContent([
  Content.multi([prompt, imagePart])
]);
```

### Получение API ключа:
1. Зайти на https://aistudio.google.com/apikey
2. Нажать "Get API key"
3. Создать ключ (бесплатно)

### Важные замечания:
- На бесплатном тарифе Google МОЖЕТ использовать данные для обучения модели
- Если нужна конфиденциальность — платный тариф ($5/мес) решает эту проблему
- Лимит 1000 запросов/день — на ВЕСЬ проект (один API ключ), не на пользователя
- Нужно реализовать очередь запросов, чтобы не превышать 15 RPM

### Сложность: Низкая
Интеграция занимает 1-2 дня. Основная работа — UI экрана камеры и отображение результатов.

## 3. Водяной знак на фото (Proof of Visit)

**Идея:** Каждое фото, сделанное сотрудником, автоматически получает штамп — доказательство, что он реально был на поле.

### Что отображается на штампе:
- GPS-координаты (широта, долгота)
- Дата и время съёмки
- Название поля / плантации
- ФИО сотрудника
- Адрес (через обратную геокодировку)

### Источник идей:
- **Mapeo Mobile** (96 stars) — https://github.com/digidem/mapeo-mobile
  Оффлайн-приложение для полевого мониторинга (React Native): GPS + фото + данные + оффлайн-синк

### Реализация на Flutter:
```dart
// Используем пакет image для наложения текста на фото
import 'package:image/image.dart' as img;

img.Image addWatermark(img.Image photo, String text) {
  img.drawString(photo, text,
    font: img.arial24,
    x: 10,
    y: photo.height - 80,
    color: img.ColorRgba8(255, 255, 255, 200),
  );
  return photo;
}
```

### Необходимые пакеты:
```yaml
image: ^4.0.0   # Обработка изображений (наложение текста)
```

## 4. Автоматический расчёт площади полигона в гектарах

**Идея:** Сотрудник нарисовал полигон на карте → моментально видит площадь в гектарах.

### Реализация:
```dart
import 'dart:math';

/// Площадь полигона на сфере (алгоритм Shoelace для сферических координат)
/// Возвращает площадь в гектарах
double calculatePolygonAreaHectares(List<LatLng> points) {
  const double earthRadius = 6371000; // метры
  double area = 0;
  int n = points.length;

  for (int i = 0; i < n; i++) {
    int j = (i + 1) % n;
    double lat1 = points[i].latitude * pi / 180;
    double lat2 = points[j].latitude * pi / 180;
    double dLng = (points[j].longitude - points[i].longitude) * pi / 180;
    area += dLng * (2 + sin(lat1) + sin(lat2));
  }

  area = (area * earthRadius * earthRadius / 2).abs();
  return area / 10000; // м² → гектары
}
```

### Необходимые пакеты:
```yaml
# Не нужно ничего нового — чистый Dart + math
```

### Сложность: Низкая (0.5 дня)
Формула уже готова, нужно только подключить к UI.

## 5. Погода для конкретного поля (Open-Meteo)

**API:** Open-Meteo — https://github.com/open-meteo/open-meteo (4 738 stars)
**Документация:** https://open-meteo.com/en/docs

### Почему Open-Meteo:
- **Полностью бесплатно** (без ключа API!)
- Без лимитов для некоммерческого использования
- Агрометеорологические данные (почва, эвапотранспирация)
- Прогноз на 16 дней

### Что показывать сотруднику для поля:
- Температура воздуха (сейчас + прогноз)
- Осадки (за неделю + прогноз)
- Температура почвы на глубине 0-7 см, 7-28 см
- Влажность почвы
- Скорость ветра
- УФ-индекс

### Пример запроса:
```
GET https://api.open-meteo.com/v1/forecast?
  latitude=41.31&longitude=69.28&
  hourly=soil_temperature_0cm,soil_moisture_0_to_1cm&
  daily=precipitation_sum,temperature_2m_max,temperature_2m_min&
  timezone=Asia/Tashkent
```

## 8. Спутниковый мониторинг здоровья поля (NDVI)

**Идея:** По полигону поля показать спутниковый снимок с индексом NDVI — зелёный = здоровое, жёлтый/красный = проблемы с растительностью.

### Источники:
| Проект | Stars | Язык | Описание |
|--------|-------|------|----------|
| **sentinelhub-py** | 885 | Python | Скачивает спутниковые снимки Sentinel-2, считает NDVI |
| **eo-learn** | 1 220 | Python | ML-фреймворк для обработки спутниковых данных |

**Репозитории:**
- https://github.com/sentinel-hub/sentinelhub-py
- https://github.com/sentinel-hub/eo-learn

### Как подключить:
1. Развернуть Python-сервер (FastAPI) на backend
2. Мобильное приложение отправляет GeoJSON-полигон поля
3. Сервер запрашивает снимок из Sentinel Hub API
4. Возвращает NDVI-карту (изображение) в приложение
5. Показать поверх карты Google Maps

### Sentinel Hub API:
- Бесплатно до 30 000 запросов/месяц
- Снимки обновляются каждые 5 дней
- Разрешение 10 м/пиксель

### Необходимые пакеты (backend):
```python
# Python (FastAPI сервер)
sentinelhub>=3.10.0
eo-learn>=1.5.0
fastapi>=0.100.0
```

### Сложность: Высокая (5-7 дней, нужен backend)
Это WOW-фича — мало кто из конкурентов её имеет. Показывает здоровье поля из космоса.
