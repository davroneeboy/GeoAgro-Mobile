# Обновление 3.3.1

## Основные изменения

### 1. Исправлено отображение статуса плантации на странице просмотра (блок "Holat")
- **Проблема:** Блок "Holat" показывал "Ko'rib chiqilmoqda" (на модерации) даже для подтверждённых плантаций, которые отображались зелёными в списке фермеров.
- **Причина:** Метод `_resolveStatus` использовал данные только из detail API (`EditPlantationModel`). Если detail API возвращал `is_checked: false`, данные из map/list API (которые корректно возвращали `is_checked: true`) полностью игнорировались.
- **Решение:** Метод `_resolveStatus` теперь объединяет данные из обоих источников через оператор `||` — если хотя бы один источник подтверждает статус, он отображается корректно.
- **Файл:** `lib/src/feature/home/view/pages/plantation_view_page.dart`

### 2. Улучшен парсинг поля `is_checked` в модели `EditPlantationModel`
- Добавлена функция `_parseBool()` для безопасного парсинга булевых значений из API.
- Обрабатывает разные типы данных: `bool`, `int` (0/1), `String` ("true"/"false").
- Заменён небезопасный каст `json["is_checked"] as bool?` на `_parseBool(json["is_checked"])`.
- **Файл:** `lib/src/data/model/plantation/edit_plantation.dart`

### 3. Разблокировано редактирование и удаление подтверждённых плантаций
- Закомментированы три проверки `isChecked`, которые:
  - Скрывали кнопки "Tahrirlash" (редактировать) и "O'chirish" (удалить) для подтверждённых плантаций
  - Блокировали удаление с сообщением "Tasdiqlangan plantatsiyani o'chirib bo'lmaydi"
  - Дублировали проверку перед самим удалением
- Теперь кнопки редактирования и удаления доступны для всех плантаций независимо от статуса.
- **Файл:** `lib/src/feature/home/view/pages/plantation_view_page.dart`

### 4. Отключена проверка количества фотографий на странице редактирования
- Закомментированы функции `calculateMinimumPhotosRequired()` и `getPhotoRequirementDetails()`, а также их вызов в валидации полей.
- На странице создания плантации (`detail_vm.dart`) проверка фотографий оставлена без изменений.
- **Файл:** `lib/src/feature/edit/vm/edit_vm.dart`

### 5. Удалён дублирующий код проверки точки в полигоне
- Удалены дублирующие публичные методы `isPointInPolygon()` и `rayCastIntersect()`.
- Единственный вызов `isPointInPolygon` в `validateCoordinatesWithLimit` заменён на существующий приватный метод `_isPointInPolygon()`, который уже использовался в трёх других местах.
- **Файл:** `lib/src/feature/google_map/vm/create_map_page_vm.dart`
