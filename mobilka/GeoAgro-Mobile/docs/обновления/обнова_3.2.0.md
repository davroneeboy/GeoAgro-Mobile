## Основные изменения

### 1. Исправление отображения статуса плантаций

#### Проблема
Плантации с `is_rejected: true` отображались как "Ko'rib chiqilmoqda" вместо "Rad etilgan".

#### Решение
- Добавлено поле `isRejected` в модели данных:
  - `Result` (lib/src/data/model/plantation/plantations_list_model.dart)
  - `RelatedPlantation` (lib/src/data/model/plantation/related_plantations_model.dart)
  - `EditPlantationModel` (lib/src/data/model/plantation/edit_plantation.dart)
  - `FarmerPlantation` (lib/src/data/model/farmer/farmer_plantation_model.dart)

- Улучшен парсинг `is_rejected` и `is_checked` из API с поддержкой разных типов данных (bool, String, int)

- Обновлена логика определения статуса:
  - Приоритет у `EditPlantationModel` (данные детальной страницы)
  - Затем `RelatedPlantation` (данные с карты)

#### Файлы
- `lib/src/data/model/plantation/plantations_list_model.dart`
- `lib/src/data/model/plantation/related_plantations_model.dart`

### 2. Изменение отображения статуса на карточках плантаций

#### Изменение
Статус плантаций теперь отображается в виде цветного индикатора (кружок) вместо текстового бейджа.

#### Детали
- Размер индикатора: 12x12 пикселей на главной странице, 10x10 пикселей в карточке фермера
- Цвета:
  - Зеленый (`success`) - "Tasdiqlangan"
  - Красный (`error`) - "Rad etilgan"
  - Оранжевый (`warning`) - "Ko'rib chiqilmoqda"
- Индикатор имеет тонкую рамку для лучшей видимости

#### Файлы
- `lib/src/feature/home/view/widgets/home_page_card_widget.dart`
- `lib/src/feature/fermers/view/widgets/farmer_plantation_card.dart`

### 3. Блокировка редактирования и удаления подтвержденных плантаций

#### Изменение
Плантации со статусом `is_checked: true` нельзя редактировать и удалять.

#### Реализация

**UI уровень:**
- На странице просмотра плантации: кнопки редактирования и удаления скрыты для подтвержденных плантаций
- На главной странице: кнопка удаления скрыта для подтвержденных плантаций
- Кнопка редактирования уже была скрыта через проверку `canEdit`

**ViewModel уровень:**
- В `deletePlantation()` добавлена проверка - блокирует удаление подтвержденных плантаций
- В `deletePlantationPermanently()` добавлена проверка - блокирует запрос на удаление подтвержденных плантаций
- При попытке удалить подтвержденную плантацию показывается сообщение: "Tasdiqlangan plantatsiyani o'chirib bo'lmaydi."

#### Файлы
- `lib/src/feature/home/view/pages/plantation_view_page.dart`
- `lib/src/feature/home/view/widgets/home_page_card_widget.dart`
- `lib/src/feature/home/vm/home_page_vm.dart`

### 4. Улучшение обработки ошибок при удалении плантаций

#### Проблема
При удалении плантации сервер возвращал ошибку в невалидном JSON формате: `{error: No Plantation matches the given query.}`, что вызывало ошибки парсинга.

#### Решение
Улучшен парсинг ошибок с поддержкой нескольких вариантов извлечения сообщения:
- `{error: message}` - с фигурными скобками
- `error: message` - без фигурных скобок
- Поиск текста после "error:" до конца строки или до закрывающей скобки

#### Файлы
- `lib/src/feature/home/vm/home_page_vm.dart`

---

### 5. Исправление ошибки "used after being disposed" в FarmersStatisticsVm

#### Проблема
При первом входе с новым токеном страница "Fermerlar" не загружалась из-за ошибки "A FarmersStatisticsVm was used after being disposed".

#### Решение
- Добавлен флаг `_isDisposed` для отслеживания состояния ViewModel
- Создан метод `_safeNotifyListeners()` для безопасного вызова `notifyListeners()`
- Добавлены проверки на `_isDisposed` во всех асинхронных методах:
  - `initialize()`
  - `fetchStatistics()`
  - `searchByInn()`
  - `clearSearch()`

#### Файлы
- `lib/src/feature/fermers/vm/farmers_statistics_vm.dart`

---

### Изменения в логике статуса

Приоритет определения статуса:
1. `EditPlantationModel.isRejected` / `EditPlantationModel.isChecked` (данные детальной страницы)
2. `RelatedPlantation.isRejected` / `RelatedPlantation.isChecked` (данные с карты)

### Защита от использования disposed ViewModel

Паттерн защиты:
```dart
bool _isDisposed = false;

void _safeNotifyListeners() {
  if (!_isDisposed) {
    notifyListeners();
  }
}

@override
void dispose() {
  _isDisposed = true;
  // ... остальной код dispose
  super.dispose();
}
```
