# Рекомендации по открытым API для страницы профиля

## 📋 Текущее состояние страницы профиля

Страница профиля (`profile_settings_page.dart`) в настоящее время отображает:
- ФИО пользователя
- Регион и район
- Номер телефона
- Ссылку на Telegram бот для поддержки

Используется API: `/api/user_info/` для получения базовой информации о пользователе.

---

## 🌐 Рекомендуемые открытые API для интеграции

### 1. **Погодные API** (Для сельскохозяйственного контекста)

#### OpenWeatherMap API
- **URL:** `https://openweathermap.org/api`
- **Бесплатный план:** 60 запросов/минуту, 1,000,000 запросов/месяц
- **Применение:**
  - Показывать текущую погоду в регионе пользователя
  - Прогноз погоды на неделю
  - Агрометеорологические данные (температура, осадки, влажность)
- **Пример использования:**
  ```dart
  // Добавить виджет погоды в профиль
  // Показывать рекомендации по уходу за плантациями на основе погоды
  ```

#### WeatherAPI.com
- **URL:** `https://www.weatherapi.com/`
- **Бесплатный план:** 1,000,000 запросов/месяц
- **Преимущества:** Поддержка агрометеорологических данных

### 2. **Статистика активности пользователя** (Используя существующие API)

#### Рекомендация: Расширить использование существующих эндпоинтов

**Доступные API в проекте:**
- `/api/plantations/forme/` - получить все плантации пользователя
- `/api/notifications/` - уведомления пользователя
- `/api/statistics/farmers` - статистика по фермерам

**Что можно добавить в профиль:**

- Последняя активность (дата последнего входа уже есть в `UserInfoModel`)
- Количество непрочитанных уведомлений

**Пример реализации:**
```dart

- Виджет статистики активности
- Счетчик плантаций
- Счетчик уведомлений
```

### 3. **Геокодирование и карты** (Расширение функциональности)

#### Nominatim (OpenStreetMap)
- **URL:** `https://nominatim.org/release-docs/develop/api/Overview/`
- **Бесплатно:** Да, без API ключа
- **Лимиты:** 1 запрос/секунду
- **Применение:**
  - Преобразование координат в адрес
  - Поиск ближайших объектов
  - Отображение карты региона пользователя

#### Google Maps Geocoding API
- **URL:** `https://developers.google.com/maps/documentation/geocoding`
- **Бесплатный план:** $200 кредитов/месяц
- **Применение:** Более точное геокодирование (если уже используется Google Maps)

## 📝 Примеры реализации

### Пример 1: Статистика плантаций в профиле

```dart
// Добавить в profile_settings_page.dart
Future<Map<String, int>> _getPlantationsStats() async {
  final response = await _repository.getPlantationsList();
  if (response != null) {
    final decoded = jsonDecode(response);
    final results = decoded['results'] as List;
    final approved = results.where((p) => p['is_checked'] == true).length;
    final pending = results.where((p) => p['is_checked'] == false).length;
    return {
      'total': results.length,
      'approved': approved,
      'pending': pending,
    };
  }
  return {'total': 0, 'approved': 0, 'pending': 0};
}
```

### Пример 2: Интеграция погодного API

```dart
// Создать новый сервис weather_service.dart
class WeatherService {
  static const String apiKey = 'YOUR_API_KEY';
  static const String baseUrl = 'https://api.openweathermap.org/data/2.5';
  
  static Future<Map<String, dynamic>> getWeather(double lat, double lon) async {
    final url = '$baseUrl/weather?lat=$lat&lon=$lon&appid=$apiKey&units=metric&lang=uz';
    final response = await http.get(Uri.parse(url));
    return jsonDecode(response.body);
  }
}
```

## 🔐 Важные замечания

1. **API ключи:** Хранить в безопасном месте (environment variables, secure storage)
2. **Лимиты запросов:** Учитывать лимиты бесплатных планов
3. **Кэширование:** Кэшировать данные для уменьшения количества запросов
4. **Обработка ошибок:** Всегда обрабатывать случаи недоступности API
5. **Офлайн режим:** Показывать кэшированные данные при отсутствии интернета
