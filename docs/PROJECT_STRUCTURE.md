# 📁 Структура проекта

## Корневая директория

```
geoagro-flutter/
├── android/           # Android нативные файлы
├── ios/              # iOS нативные файлы
├── lib/              # Dart код приложения
├── test/             # Тесты
├── assets/           # Ресурсы (изображения, SVG, Lottie)
├── api-docs/         # Документация API и спецификации
├── docs/             # Общая документация проекта
├── scripts/          # Скрипты разработки (игнорируются в git)
├── logs/             # Логи приложения (игнорируются в git)
├── tmp/              # Временные файлы (игнорируются в git)
├── server/           # Серверные файлы (Node.js update server, игнорируются в git)
├── pubspec.yaml      # Зависимости Flutter
├── firebase.json     # Конфигурация Firebase
└── README.md         # Основная документация
```

## 📂 Описание папок

### `/scripts/` - Скрипты разработки
Содержит все вспомогательные bash скрипты для разработки:
- `run_emulator_with_logs.sh` - ⭐ **Основной скрипт для запуска эмулятора с логами**
- `start_dev.sh` - Запуск с auto hot reload
- `show_logs.sh` - Просмотр live логов
- `watch_logs.sh` - Просмотр всех логов Flutter
- `watch_dio_logs.sh` - Просмотр HTTP запросов/ответов
- И другие вспомогательные скрипты

📖 Подробнее: [scripts/README.md](../scripts/README.md)

### `/docs/` - Документация
- `START_APP.md` - Инструкции по запуску приложения
- `PROJECT_STRUCTURE.md` - Этот файл

### `/api-docs/` - API документация
- `MOBILE_SPECIAL_USER_FEATURES.md` - Спецификация функций для special users

### `/logs/` - Логи
Автоматически создаваемые логи приложения. Игнорируются в git.

### `/tmp/` - Временные файлы
Временные файлы разработки. Игнорируются в git.

### `/server/` - Серверные файлы
Node.js сервер для обновлений приложения:
- `package.json` - Зависимости Node.js
- `update_server_example.js` - Пример сервера обновлений
- `render_package.json` - Конфигурация для Render

Игнорируются в git.

## 🚀 Быстрый старт

### Запуск эмулятора с логами
```bash
./scripts/run_emulator_with_logs.sh
```

### Просмотр логов
```bash
./scripts/show_logs.sh
```

### Запуск приложения
```bash
flutter run -d emulator-5554
```

## 📝 Правила организации

1. **Все скрипты разработки** должны быть в `/scripts/`
2. **Вся документация** должна быть в `/docs/` или `/api-docs/`
3. **Логи и временные файлы** автоматически игнорируются git
4. **Серверные файлы** находятся в `/server/` и игнорируются git
